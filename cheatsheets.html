Quick Flow

Typical Commit to GitHub Sequence

01

Inspect

Check branch and changed files before staging anything.

git status
02

Stage

Add exactly what belongs in the next commit.

git add <file>
03

Commit

Write one clear message that explains the change.

git commit -m "fix auth redirect"
04

Push

Send the branch to GitHub and set upstream when needed.

git push -u origin feature/auth-fix
05

Open PR

Create a pull request from the command line if you use GitHub CLI.

gh pr create

Basics

Inspect and Stage

Use these before every commit to keep diffs clean.

Working tree status

git status

Shows modified, staged, and untracked files on the current branch.

Stage one file

git add src/app.ts

Add one file to the next commit without staging unrelated work.

Stage everything tracked

git add -A

Stages all tracked and untracked changes in the repo.

Review staged diff

git diff --cached

Confirms exactly what will be included in the commit.

Commits

Create and Adjust Commits

Common commands for clean commit history without leaving the terminal.

Make a commit

git commit -m "add profile save validation"

Creates a commit with a short, direct message.

Amend last commit

git commit --amend

Edits the last commit message or includes newly staged changes.

Amend without changing message

git commit --amend --no-edit

Useful when you forgot one file and want to keep the same message.

See recent commits

git log --oneline --decorate -n 8

Quick compact view of the latest local history.

Branches

Branch and Push to GitHub

Use a branch per change, then publish it with upstream tracking.

Create and switch branch

git switch -c feature/commit-cheatsheet

Creates a new branch and moves you onto it immediately.

Push current branch first time

git push -u origin feature/commit-cheatsheet

Publishes the branch and sets upstream so later pushes can be shorter.

Push after upstream exists

git push

Sends new commits to the tracked remote branch.

Show configured remotes

git remote -v

Checks where `origin` and other remotes point before you push.

Sync

Stay in Sync with Remote

Fetch before risky work and rebase or pull with intent.

Fetch latest remote data

git fetch origin

Updates remote-tracking branches without modifying your working tree.

Rebase onto main

git rebase origin/main

Replays your branch on top of the latest remote main branch.

Pull with rebase

git pull --rebase

Updates the current branch while keeping history linear.

Force push after rebase

git push --force-with-lease

Safer than `--force` because it refuses to overwrite unexpected remote work.

GitHub CLI

Open Pull Requests from Terminal

Useful if you use `gh` for GitHub-native actions after pushing commits.

Authenticate GitHub CLI

gh auth login

Connects the local machine to your GitHub account.

Create a pull request

gh pr create --fill

Opens a PR and auto-fills title and body from commit context when possible.

Check PR status

gh pr status

Shows your current pull requests, review requests, and merge state.

Open repo in browser

gh repo view --web

Jumps straight to the repository page on GitHub.

Message Pattern

Useful Commit Style

Keep the first line short and action-oriented. Good examples:

  • fix login redirect loop
  • add markdown download button
  • refactor blog index rendering

Safe Default

Before You Push

Run this quick review loop if the branch matters:

  • git status
  • git diff --cached
  • git log --oneline --decorate -n 3

Arch Linux

Safe System Update Sequence

01

Read News

Check for manual interventions before major upgrades.

xdg-open https://archlinux.org/news/
02

Sync Keys

Refresh the keyring first if signatures fail or the install is stale.

sudo pacman -Sy archlinux-keyring
03

Upgrade

Perform a full system upgrade; avoid partial upgrades.

sudo pacman -Syu
04

Review

Check failed services and recent boot logs after the update.

systemctl --failed
05

Clean

Remove orphaned packages only after confirming the list.

pacman -Qdtq

Pacman

Packages and Updates

Use full upgrades, inspect packages before removing them, and avoid partial upgrade states.

Full system upgrade

sudo pacman -Syu

Synchronizes repositories and upgrades every installed package.

Install package

sudo pacman -S <package>

Installs a package from enabled repositories.

Search repositories

pacman -Ss <term>

Searches package names and descriptions.

Package details

pacman -Qi <package>

Shows installed package metadata, dependencies, and install reason.

Remove package and unused deps

sudo pacman -Rns <package>

Removes the package, unused dependencies, and config files tracked by pacman.

Find orphans

pacman -Qdtq

Lists dependency packages no longer required by installed packages.

AUR

Build User Repository Packages

Review PKGBUILDs before building; AUR packages are user-submitted and not official repo packages.

Install build tools

sudo pacman -S --needed base-devel git

Installs the standard toolchain needed to build packages.

Clone package build files

git clone https://aur.archlinux.org/<package>.git

Downloads the AUR package recipe for local review and build.

Inspect the build recipe

less PKGBUILD

Read what the package downloads, builds, and installs before running it.

Build and install

makepkg -si

Builds the package and installs missing dependencies from the repositories.

Systemd

Services and Logs

Inspect service state before changing startup behavior, then use the journal for failure context.

Show failed units

systemctl --failed

Lists services and units that failed during the current boot.

Service status

systemctl status <unit>

Shows whether a service is active, failed, disabled, or waiting.

Enable and start now

sudo systemctl enable --now <unit>

Starts the service immediately and enables it for future boots.

Current boot logs

journalctl -b -p warning

Shows warnings and errors from the current boot.

Logs for one service

journalctl -u <unit> -b

Filters the current boot journal to a single systemd unit.

Network

Connectivity and DNS

Check link state, routes, DNS, and NetworkManager before blaming applications.

Interfaces and addresses

ip addr

Lists network interfaces, addresses, and link state.

Routes

ip route

Shows the default gateway and routing table.

DNS state

resolvectl status

Displays DNS servers and per-link resolver configuration.

NetworkManager devices

nmcli device status

Shows managed devices and connection state when using NetworkManager.

Connectivity test

ping -c 4 archlinux.org

Sends four ICMP packets to verify basic network reachability.

Storage

Disks, Mounts, and Space

Use filesystem-aware commands before mounting, resizing, or cleaning storage.

Block devices

lsblk -f

Shows disks, partitions, filesystems, labels, UUIDs, and mount points.

Mounted filesystems

findmnt

Displays the mount tree and filesystem sources.

Disk usage

df -h

Shows mounted filesystem capacity in human-readable units.

Directory size scan

du -sh * 2>/dev/null

Summarizes size by directory in the current location.

Mount a partition

sudo mount /dev/<partition> /mnt

Mounts a partition for repair, install, backup, or recovery work.

Users

Accounts and Sudo

Confirm identity and groups before changing permissions or sudo access.

Current user

whoami

Prints the active username for the current shell.

User and groups

id

Shows UID, GID, and group membership.

Add user to wheel

sudo usermod -aG wheel <user>

Adds a user to the common admin group used by sudoers examples.

Change password

passwd

Changes the current user's password.

Edit sudoers safely

sudo visudo

Checks sudoers syntax before saving changes.

Recovery

Repair and Maintenance

Useful commands for stale keys, bootloader regeneration, initramfs rebuilds, and cache cleanup.

Refresh Arch keyring

sudo pacman -Sy archlinux-keyring

Updates package signing keys before a full upgrade.

Verify installed files

sudo pacman -Qkk

Checks installed package files for missing or modified paths.

Rebuild initramfs

sudo mkinitcpio -P

Regenerates initramfs images for all configured presets.

Regenerate GRUB config

sudo grub-mkconfig -o /boot/grub/grub.cfg

Writes a fresh GRUB configuration after kernel or boot changes.

Clean package cache

sudo pacman -Sc

Removes old package cache entries after you no longer need rollback copies.

Arch Rule

Avoid Partial Upgrades

Do not install a single package from a freshly synced database without upgrading the rest of the system. Prefer sudo pacman -Syu before installing new software.

AUR Habit

Review Before Build

AUR package files can run arbitrary build steps. Read PKGBUILD and related install files before using makepkg -si.
