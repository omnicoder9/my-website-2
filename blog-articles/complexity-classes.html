Some problems yield to cleverness. Others seem to resist every clever idea anyone has ever had — not because we lack a method, but because the difficulty appears to be woven into the problem itself. Computational complexity theory is the discipline that tries to make this intuition exact: to draw, with mathematical rigor, the line between the tractable and the intractable. Its central vocabulary is a small family of complexity classes — P, NP, co-NP, and the notions of NP-hardness and NP-completeness — and the relationships between them encode one of the deepest open questions in all of science.

That question, does P equal NP?, was named one of the seven Millennium Prize Problems by the Clay Mathematics Institute in 2000, each carrying a one-million-dollar reward for a correct solution.6 To date only one of those seven has been solved, and P versus NP is not among them. This article builds up the machinery class by class, grounds each definition in its original source, and shows how a single problem about Boolean formulas came to organize the entire territory.

Section 01What a complexity class measures

Complexity theory studies decision problems: questions with a yes-or-no answer, posed over inputs of varying size. "Is this graph 3-colorable?" and "Is this number prime?" are decision problems; the size of an input is, roughly, the number of bits needed to write it down. A complexity class is then a collection of decision problems that share a resource bound — typically a bound on the time a Turing machine needs to decide them, measured as a function of input size.

The key modeling decision, made in the mid-1960s, was to treat polynomial time as the formal stand-in for "efficiently solvable." Alan Cobham's 1965 paper on the intrinsic computational difficulty of functions is among the earliest to isolate the class of polynomial-time-decidable problems as a natural object,2 and in the same year Jack Edmonds, describing a polynomial-time algorithm for maximum matching, explicitly argued that polynomial running time is what separates a "good" algorithm from an impractical one.3 The resulting principle — that a problem is feasibly computable exactly when it lies in polynomial time — is now called the Cobham–Edmonds thesis.3

A caveat worth keeping

"Polynomial = efficient" is an idealization. An algorithm running in n¹⁰⁰ steps is polynomial yet useless in practice, while some exponential-worst-case algorithms run quickly on real instances. The identification is a theoretical convenience that has proven extraordinarily fruitful — but it is a thesis about modeling, not a theorem.3

Section 02The class P — solvable quickly

P (for polynomial time) is the set of decision problems that a deterministic Turing machine can decide in a number of steps bounded by some polynomial in the input size. Formally, a problem is in P if there is an algorithm and a constant c such that every n-bit instance is decided within O(nᶜ) steps, where c depends on the problem but not the instance.1

Class definition
P — Polynomial Time

Decision problems decidable by a deterministic Turing machine in polynomial time. Intuitively: the problems we can actually solve efficiently.

P = ⋃c≥1 DTIME(n^c)

P is populated by the workhorses of computing: shortest paths, sorting, linear programming, matching, and so on. Membership in P is not always obvious. A celebrated example is primality testing. Whether a given integer is prime was long known to be decidable, and fast randomized tests existed, but a deterministic polynomial-time algorithm remained elusive until 2002, when Agrawal, Kayal, and Saxena published "PRIMES is in P," settling the question and demonstrating that membership in P can itself be a hard-won mathematical result.15

Section 03The class NP — checkable quickly

Now relax the demand. Instead of requiring that we find a solution efficiently, require only that, given a candidate solution, we can check it efficiently. This is the essence of NP. The Clay Institute frames the whole subject around exactly this asymmetry: if it is easy to verify that a proposed solution is correct, must it also be easy to find one?7

There are two equivalent definitions, and holding both in mind is the surest route to understanding the class. The first is operational: NP is the set of problems decidable in polynomial time by a nondeterministic Turing machine — a hypothetical device that may, at each step, branch into multiple possibilities and accepts if any branch does. The second is the certificate view: a problem is in NP if every yes-instance has a short (polynomial-length) "certificate" or "witness" that a deterministic verifier can check in polynomial time.16

Class definition
NP — Nondeterministic Polynomial Time

Decision problems whose yes-instances admit a polynomial-size certificate verifiable in polynomial time — equivalently, those decidable in polynomial time by a nondeterministic Turing machine.

x ∈ L ⇔ ∃ c, |c| ≤ poly(|x|), with V(x,c) accepting in poly time

The Clay Institute's chosen illustration is the Hamiltonian path problem: given a list of cities, is there a route that visits each exactly once? Hand someone a proposed route and they can verify it almost instantly; finding one from scratch is another matter entirely.7 Sudoku, graph coloring, and the Boolean satisfiability problem all share this shape — solutions easy to check, seemingly hard to construct.

Crucially, every problem in P is also in NP. If you can solve a problem outright in polynomial time, you can certainly verify a proposed answer in polynomial time (just ignore the certificate and solve it). So P ⊆ NP is immediate. The entire drama lies in whether that containment is strict.

Section 04P versus NP — the question itself

The question P = NP? asks whether efficient verification implies efficient solution. If P = NP, then for every problem whose answers we can check quickly, we can also find those answers quickly — a world in which much of cryptography collapses, optimization becomes routine, and, as Lance Fortnow puts it in his survey of the problem's status, short formal proofs of theorems could in principle be discovered automatically.11 If instead P ≠ NP, the asymmetry is real: some problems are genuinely harder to solve than to check.

The problem was crystallized in 1971 by Stephen Cook in a paper presented at the third ACM Symposium on Theory of Computing,1 and independently formulated around the same time by Leonid Levin in the Soviet Union, working on what he called universal search problems.5 Their parallel discovery is why the foundational result discussed below carries both names.

How do experts bet? Since 2002, William Gasarch has polled the theory community three times. The proportion expecting P ≠ NP rose from 61% in 2002 (100 respondents) to 83% in 2012 (152 respondents) to 88% in 2019 (124 respondents); among researchers judged to have thought hardest about the problem, the 2019 figure reaches roughly 99%.121314 Gasarch is careful to note that a poll measures opinion, not truth — it "does not bring us any closer to solving" the question.12

Section 05Mapping the landscape

Before going further it helps to see the territory. The diagram below contrasts the two possible worlds. On the left, the world most theorists expect — P ≠ NP — where NP splits into three disjoint regions: the easy problems (P), the hardest problems (NP-complete), and a strange in-between (NP-intermediate). On the right, the world where P = NP and these distinctions evaporate.

Section 06NP-hard and NP-complete — the hardest problems

To compare difficulty, complexity theory uses reductions. A polynomial-time reduction from problem A to problem B is an efficient transformation of any instance of A into an instance of B with the same yes/no answer. If such a reduction exists, then B is "at least as hard as" A: any fast algorithm for B yields a fast algorithm for A.

This gives two precise notions of hardness:

Class definition
NP-hard — at least as hard as all of NP

A problem to which every problem in NP reduces in polynomial time. NP-hard problems need not be in NP themselves, and need not even be decision problems.8

Class definition
NP-complete — the hardest problems inside NP

A problem that is both in NP and NP-hard. These are the maximal-difficulty members of NP: if any one of them is in P, then all of NP is in P.8

L is NP-complete ⇔ L ∈ NP ∧ L is NP-hard

The defining leverage of NP-completeness is this: the NP-complete problems stand or fall together. Because every NP problem reduces to each of them, a polynomial-time algorithm for a single NP-complete problem would place all of NP into P — proving P = NP. Equivalently, if even one NP-complete problem can be shown to lie outside P, then P ≠ NP. The thousands of known NP-complete problems are, in this sense, a single problem wearing thousands of disguises.8

An NP-complete problem is at least as tough as any other problem in NP — solve one efficiently and you have solved them all.

Section 07Cook–Levin and Karp's cascade

For NP-completeness to be more than a definition, someone had to exhibit a first NP-complete problem — a problem provably at least as hard as everything in NP without circular appeal to another complete problem. Cook supplied it in 1971. The Cook–Levin theorem states that the Boolean satisfiability problem, SAT — given a Boolean formula, is there an assignment of truth values making it true? — is NP-complete: it lies in NP, and every problem in NP can be reduced to it in polynomial time by a deterministic Turing machine.15

The proof is a tour de force of encoding: Cook showed that the entire computation of any polynomial-time nondeterministic machine on a given input can be captured as a Boolean formula that is satisfiable precisely when the machine accepts. Once SAT is anchored, every further NP-completeness proof becomes comparatively easy — reduce SAT (or a problem already known complete) to the new target.

That is exactly what Richard Karp did in 1972. In "Reducibility Among Combinatorial Problems," he used Cook's result to give polynomial-time reductions establishing that 21 classic combinatorial and graph problems are NP-complete, among them clique, vertex cover, the Hamiltonian circuit, the knapsack problem, and integer programming.4 Karp's paper was the moment the field saw that NP-completeness was not a curiosity but a pervasive feature of computation — that intractability lurks across optimization, scheduling, logic, and graph theory alike.4 The canonical reference cataloguing the resulting theory, Garey and Johnson's Computers and Intractability, remains the standard guide decades later.8

The reduction chain, in miniature

SAT → 3-SAT → CLIQUE → VERTEX COVER → … Each arrow is a polynomial-time transformation preserving the answer. Because the chain originates at SAT — which absorbs all of NP via Cook–Levin — every problem along it inherits NP-completeness. New problems are typically proven NP-complete by extending this chain with one more reduction.4

Section 08co-NP — certifying the "no"

NP is, in a subtle way, lopsided. Its certificates witness yes-instances: a satisfying assignment proves a formula is satisfiable. But what certifies a no? To prove a formula is unsatisfiable, exhibiting one failed assignment is hopeless — you would have to rule out all of them. This asymmetry motivates a mirror class.

Class definition
co-NP — efficiently certifiable "no" answers

The set of decision problems whose complements are in NP. Equivalently, problems for which every no-instance has a short certificate checkable in polynomial time.16

L ∈ co-NP ⇔ (complement of L) ∈ NP

The complement of SAT is the canonical co-NP-complete problem: TAUTOLOGY — is a given formula true under every assignment? — together with UNSAT, the unsatisfiability problem.17 Reversing the "accept" and "reject" labels of an NP-complete problem produces a co-NP-complete one.17

Two structural facts orient co-NP relative to its neighbors. First, P sits inside both NP and co-NP: a problem solvable in polynomial time can have its complement solved in polynomial time too, so P ⊆ NP ∩ co-NP.16 Second — and this is another major open problem — it is not known whether NP = co-NP. Most theorists believe they differ; Edmonds himself, back in the 1960s, conjectured both that NP ≠ P and that NP ∩ co-NP = P.3 If NP ≠ co-NP, then P ≠ NP follows immediately, since P = NP would force the two to coincide.

Section 09The space between — NP-intermediate problems

Suppose P ≠ NP. A tempting picture is that NP then cleanly divides into the easy (P) and the maximally hard (NP-complete), with nothing in between. Ladner's theorem demolishes this tidy hope. In 1975 Richard Ladner proved that if P ≠ NP, then NP necessarily contains problems that are neither in P nor NP-complete — the NP-intermediate class, NPI, is non-empty.9

The construction is a diagonalization that builds an artificial problem deliberately tuned to evade both P and NP-completeness; the language is, in Arora and Barak's words, otherwise uninteresting.16 What makes the result striking is its converse implication: NPI is empty if and only if P = NP. Whether any natural problem is NP-intermediate is open, but the long-standing candidates are evocative — the graph isomorphism problem and the decision versions of integer factorization and discrete logarithm, none of which has ever been placed in P or proven NP-complete.9 The presumed hardness of factoring is, not coincidentally, the foundation on which much of public-key cryptography rests.

Section 10Why the proof resists

If almost everyone believes P ≠ NP, why has no one proved it after half a century? Part of the answer is that the most natural proof techniques are provably too weak. The first formal "barrier" came in 1975, when Baker, Gill, and Solovay studied the question relative to oracles — hypothetical subroutines that solve some fixed problem in a single step. They constructed one oracle A under which PA = NPA, and another oracle B under which PB ≠ NPB.10

The consequence is sharp. Any proof technique that would work equally well in the presence of any oracle — the property called relativization — cannot settle P versus NP, because the answer itself flips depending on the oracle. Diagonalization, the technique that resolves many other separation questions, relativizes, and is therefore ruled out as a complete solution.11 Fortnow's survey walks through this and the later algebrization and natural-proofs barriers, each of which closes off another avenue and helps explain why a resolution likely requires fundamentally new mathematics.11

Section 11Why any of this matters

The stakes are not academic. A constructive proof that P = NP — an actual efficient algorithm for an NP-complete problem — would break the cryptographic schemes that secure essentially all digital communication, since their security rests on certain problems being hard to solve but easy to verify. It would also transform optimization, logistics, protein folding, circuit design, and automated theorem proving, collapsing problems we currently attack with heuristics into routine computation.11

The widely expected outcome, P ≠ NP, is in its own way the more reassuring: it would confirm that the hardness we perceive is real, that some problems are intrinsically harder to solve than to check, and that the security guarantees we depend on rest on solid mathematical ground rather than on collective failure of imagination. Either way, the classes in this article are not bureaucratic categories. They are the coordinate system on which the limits of efficient computation are drawn — and the boundary between two of them remains the most consequential blank space on the map.
