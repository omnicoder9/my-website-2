Introduction

The Core Insight

The arrays-and-hashing stage teaches a single transferable reflex: a nested loop that searches for something inside an array can almost always be collapsed into a single pass by pre-loading what we need into a hash map or set before—or during—the scan. What was O(n²) becomes O(n); what required a sort becomes a one-pass traversal.

This is not a bag of tricks. It is one principle applied six different ways. Every problem below is a variation on the same question: what do I need at each position, and can I answer that in O(1) instead of O(n)?

The problems in this section are largely rated Easy to Medium, but they are not easy to use well under contest pressure. The skill you are building is pattern recognition—seeing the underlying structure quickly enough to reach for the right tool before your opponents do.

When a Hash Map Applies

Three signals tell you a hash map will help:

  • 1. The inner loop is searching for a specific value (complement, duplicate, group key).
  • 2. You need to count how many times each value has appeared so far.
  • 3. You need to group items that share a canonical property (anagram key, frequency signature).

When the value domain is small and known (e.g. lowercase letters, bytes 0–255), prefer a fixed-size array as a perfect hash: it is simpler, faster in practice, and requires no collision handling.

Contains DuplicateLC 217

EasyHash SetArray
Naïve
O(n²)
O(1) space
→
Optimal
O(n)
O(n) space
Key insightA set gives O(1) amortised membership testing. One pass is all we need: attempt to insert each element; if the insert fails (element already present), we have our duplicate.

Problem Statement

Given an integer array nums, return true if any value appears at least twice, and false if every element is distinct.

Example

Input
[1,2,3,1]
4 elements
Output
true
value 1 repeats

Approach

Maintain a hash set of seen values. For each element, check membership before insertion. If the element is already present, return true immediately. If we exhaust the array without finding a duplicate, return false. The early-exit means best-case performance is O(1) (first two elements identical); worst case O(n) when all elements are distinct.

Note the equivalent idiomatic shortcut available in most languages: HashSet.add() returns false when insertion fails (Java, C#), and Rust's HashSet::insert returns false on collision. These eliminate the explicit pre-check.

func containsDuplicate(nums []int) bool {
    seen := make(map[int]bool)
    for _, n := range nums {
        if seen[n] {
            return true
        }
        seen[n] = true
    }
    return false
}
public boolean containsDuplicate(int[] nums) {
    Set<Integer> seen = new HashSet<>();
    for (int n : nums) {
        if (!seen.add(n)) return true;
    }
    return false;
}
bool containsDuplicate(vector<int>& nums) {

    for (int n : nums) {
        if (seen.count(n)) return true;
        seen.insert(n);
    }
    return false;
}

Valid AnagramLC 242

EasyFrequency ArrayString
Naïve
O(n log n)
O(1) space
→
Optimal
O(n)
O(1) space
Key insightA 26-element integer array is a perfect hash map for lowercase letters. Index = char − 'a'. Increment for s, decrement for t; any non-zero cell at the end reveals a mismatch. Zero additional allocation beyond the fixed 26 ints.

Problem Statement

Given two strings s and t, return true if t is an anagram of s, and false otherwise. An anagram is a rearrangement of all characters of a string.

Example

s
anagram
t
nagaram
Output
true
same char frequencies

Approach

An early length check eliminates mismatches immediately. Then we single-pass both strings simultaneously, incrementing the count for each character in s and decrementing for the corresponding character in t. At the end, all 26 counters must be zero for the strings to be anagrams. The single allocation is a 26-element integer array—no dynamic memory, no hash collision handling.

For strings over arbitrary Unicode, replace the array with a HashMap<char, int>. Time complexity stays O(n); space becomes O(k) where k is the number of distinct characters (at most min(n, alphabet size)).

func isAnagram(s string, t string) bool {
    if len(s) != len(t) {
        return false
    }
    count := [26]int{}
    for i := 0; i < len(s); i++ {
        count[s[i]-'a']++
        count[t[i]-'a']--
    }
    for _, c := range count {
        if c != 0 {
            return false
        }
    }
    return true
}
public boolean isAnagram(String s, String t) {
    if (s.length() != t.length()) return false;
    int[] count = new int[26];
    for (int i = 0; i < s.length(); i++) {
        count[s.charAt(i) - 'a']++;
        count[t.charAt(i) - 'a']--;
    }
    for (int c : count) {
        if (c != 0) return false;
    }
    return true;
}

pub fn is_anagram(s: String, t: String) -> bool {
    if s.len() != t.len() { return false; }
    let mut count = [0i32; 26];
    for (a, b) in s.bytes().zip(t.bytes()) {
        count[(a - b'a') as usize] += 1;
        count[(b - b'a') as usize] -= 1;
    }
    count.iter().all(|&c| c == 0)
}
public bool IsAnagram(string s, string t) {

        count[t[i] - 'a']--;
    }
    return count.All(c => c == 0);
}
function isAnagram(s: string, t: string): boolean {
    if (s.length !== t.length) return false;
    const count = new Array(26).fill(0);
    for (let i = 0; i < s.length; i++) {
        count[s.charCodeAt(i) - 97]++;
        count[t.charCodeAt(i) - 97]--;
    }
    return count.every(c => c === 0);
}

  • Find All Anagrams in a String (LC 438) — sliding window of width p over s; re-use frequency array with single-character updates per slide.

Two SumLC 1

EasyHash MapComplement Lookup
Naïve
O(n²)
O(1) space
→
Optimal
O(n)
O(n) space
Key insightReformulate: for element x, we need target − x. Store each element's index as we scan, and look up the complement before storing x. Because we look up before inserting, we never match an element against itself.

Problem Statement

Given an integer array nums and an integer target, return the indices of the two numbers such that they add up to target. Exactly one solution is guaranteed; you may not use the same element twice.

Example

nums
[2,7,11,15]
target = 9
Output
[0,1]
2 + 7 = 9

Approach

For each element nums[i], compute complement = target − nums[i]. If the complement was already seen (i.e. it is in our map), we have our answer. Otherwise, record nums[i] → i in the map and continue.

Critical ordering: we look up before storing. This ensures we never pair an element with itself, even when nums[i] * 2 == target. The problem guarantees exactly one valid pair, so we can return immediately on the first match.

func twoSum(nums []int, target int) []int {
    seen := make(map[int]int)
    for i, n := range nums {
        complement := target - n
        if j, ok := seen[complement]; ok {
            return []int{j, i}
        }
        seen[n] = i
    }
    return nil
}
public int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> seen = new HashMap<>();
    for (int i = 0; i < nums.length; i++) {
        int complement = target - nums[i];
        if (seen.containsKey(complement)) {
            return new int[]{seen.get(complement), i};
        }
        seen.put(nums[i], i);
    }
    return new int[]{};
}
vector<int> twoSum(vector<int>& nums, int target) {
    unordered_map<int, int> seen;
    for (int i = 0; i < (int)nums.size(); i++) {
        int comp = target - nums[i];
        if (seen.count(comp)) return {seen[comp], i};
        seen[nums[i]] = i;
    }
    return {};
}
use std::collections::HashMap;

pub fn two_sum(nums: Vec<i32>, target: i32) -> Vec<i32> {
    let mut seen: HashMap<i32, i32> = HashMap::new();
    for (i, &n) in nums.iter().enumerate() {
        let complement = target - n;
        if let Some(&j) = seen.get(&complement) {
            return vec![j, i as i32];
        }
        seen.insert(n, i as i32);
    }
    vec![]
}
public int[] TwoSum(int[] nums, int target) {
    var seen = new Dictionary<int, int>();
    for (int i = 0; i < nums.Length; i++) {
        int complement = target - nums[i];
        if (seen.TryGetValue(complement, out int j)) {
            return new int[] { j, i };
        }
        seen[nums[i]] = i;
    }
    return Array.Empty<int>();
}
function twoSum(nums: number[], target: number): number[] {
    const seen = new Map<number, number>();
    for (let i = 0; i < nums.length; i++) {
        const complement = target - nums[i];
        if (seen.has(complement)) return [seen.get(complement)!, i];
        seen.set(nums[i], i);
    }
    return [];
}

Variations & Extensions

  • Two Sum II (LC 167) — sorted input: two-pointer O(n) time, O(1) space.
  • Three Sum (LC 15) — sort + fix one element + two-pointer inner loop; O(n²) total.
  • Four Sum (LC 18) — add another fix layer; O(n³). Hash map variant can shave a factor but adds complexity.
  • Two Sum BST (LC 653) — given two BSTs, find pair with given sum: in-order traversal + set.

Group AnagramsLC 49

MediumCanonical KeyGrouping
Naïve
O(n · k log k)
O(nk) space
→
Optimal
O(n · k)
O(nk) space
Key insightTwo strings are anagrams iff they share a canonical form. Sorting gives one canonical form in O(k log k); a 26-element frequency tuple gives another in O(k). The tuple approach is strictly faster for long strings and is the preferred competition solution.

Problem Statement

Given an array of strings strs, group the anagrams together and return them in any order.

Example

Input
["eat","tea","tan","ate","nat","bat"]
6 words
Output
[["bat"],["nat","tan"],["ate","eat","tea"]]
3 groups

Approach

Anagrams are equivalent under character rearrangement, so we need a canonical form that collapses all anagrams of a word to the same key. Two canonical forms are common:

Sort-based key: sort each string's characters → "eat","tea","ate" all become "aet". O(k log k) per word.

Frequency-tuple key: compute the 26-element frequency array → [1,0,0,0,1,0,...,1,0,0] for all anagrams of "eat". O(k) per word. This is the preferred approach for long strings.

Go's fixed-size array [26]int is hashable as a map key natively—a uniquely elegant solution. Other languages must serialize the array (Java: Arrays.toString, TypeScript: join(',')).

func groupAnagrams(strs []string) [][]string {
    groups := make(map[[26]int][]string)
    for _, s := range strs {
        var key [26]int
        for _, c := range s {
            key[c-'a']++
        }
        groups[key] = append(groups[key], s)
    }
    result := make([][]string, 0, len(groups))
    for _, g := range groups {
        result = append(result, g)
    }
    return result
}
public List<List<String>> groupAnagrams(String[] strs) {
    Map<String, List<String>> groups = new HashMap<>();
    for (String s : strs) {
        int[] count = new int[26];
        for (char c : s.toCharArray()) count[c - 'a']++;
        String key = Arrays.toString(count);
        groups.computeIfAbsent(key, k -> new ArrayList<>()).add(s);
    }
    return new ArrayList<>(groups.values());
}
vector<vector<string>> groupAnagrams(vector<string>& strs) {
    map<array<int,26>, vector<string>> groups;
    for (auto& s : strs) {
        array<int,26> key{};
        for (char c : s) key[c - 'a']++;
        groups[key].push_back(s);
    }
    vector<vector<string>> res;
    for (auto& [k, v] : groups) res.push_back(v);
    return res;
}
use std::collections::HashMap;

pub fn group_anagrams(strs: Vec<String>) -> Vec<Vec<String>> {
    let mut groups: HashMap<[u8; 26], Vec<String>> = HashMap::new();
    for s in strs {
        let mut key = [0u8; 26];
        for b in s.bytes() {
            key[(b - b'a') as usize] += 1;
        }
        groups.entry(key).or_default().push(s);
    }
    groups.into_values().collect()
}
public IList<IList<string>> GroupAnagrams(string[] strs) {
    var groups = new Dictionary<string, List<string>>();
    foreach (var s in strs) {
        int[] count = new int[26];
        foreach (char c in s) count[c - 'a']++;
        string key = string.Join(",", count);
        if (!groups.ContainsKey(key)) groups[key] = new List<string>();
        groups[key].Add(s);
    }
    return groups.Values.Cast<IList<string>>().ToList();
}
function groupAnagrams(strs: string[]): string[][] {
    const groups = new Map<string, string[]>();
    for (const s of strs) {
        const count = new Array(26).fill(0);
        for (const c of s) count[c.charCodeAt(0) - 97]++;
        const key = count.join(',');
        if (!groups.has(key)) groups.set(key, []);
        groups.get(key)!.push(s);
    }
    return Array.from(groups.values());
}

Variations & Extensions

  • Sort-based key — simpler to write; replace the 26-int key with sort(s). Correct but slower for long strings.
  • Count distinct anagram groups — just return groups.size().
  • Largest anagram group — keep track of max group size during construction.
  • Find All Anagrams in a String (LC 438) — sliding window variant of the same frequency-key idea.

Top K Frequent ElementsLC 347

MediumBucket SortFrequency Map
Naïve
O(n log n)
O(n) space
→
Optimal
O(n)
O(n) space
Key insightAn element's frequency is at most n (all elements equal). So we can allocate n+1 buckets, indexed by frequency, in O(n). Walking buckets from highest frequency downward then yields the top-k in O(n) without any comparison sort.

Problem Statement

Given an integer array nums and an integer k, return the k most frequent elements. The answer may be returned in any order.

Example

nums
[1,1,1,2,2,3]
k = 2
Output
[1,2]
freq: 1→3, 2→2, 3→1

Approach

The naïve heap-of-size-k approach runs in O(n log k). We can do better by exploiting the constraint: an element's frequency is at most n. So create an array of n+1 buckets, where bucket[f] holds all elements appearing exactly f times. Walk buckets from index n down to 0, collecting elements until we have k. Total work: O(n) for both the frequency pass and the bucket walk.

This is bucket sort applied to frequency rather than value—the same trick that makes counting sort O(n) for bounded-range integers.

func topKFrequent(nums []int, k int) []int {
    freq := make(map[int]int)
    for _, n := range nums {
        freq[n]++
    }
    // bucket[i] = numbers that appear exactly i times
    buckets := make([][]int, len(nums)+1)
    for num, count := range freq {
        buckets[count] = append(buckets[count], num)
    }
    result := make([]int, 0, k)
    for i := len(buckets) - 1; i >= 0 && len(result) < k; i-- {
        result = append(result, buckets[i]...)
    }
    return result[:k]
}
public int[] topKFrequent(int[] nums, int k) {
    Map<Integer, Integer> freq = new HashMap<>();
    for (int n : nums) freq.merge(n, 1, Integer::sum);

    @SuppressWarnings("unchecked")
    List<Integer>[] buckets = new List[nums.length + 1];
    for (var entry : freq.entrySet()) {
        int cnt = entry.getValue();
        if (buckets[cnt] == null) buckets[cnt] = new ArrayList<>();
        buckets[cnt].add(entry.getKey());
    }
    int[] result = new int[k];
    int idx = 0;
    for (int i = buckets.length - 1; i >= 0 && idx < k; i--) {
        if (buckets[i] != null)
            for (int num : buckets[i])
                if (idx < k) result[idx++] = num;
    }
    return result;
}
vector<int> topKFrequent(vector<int>& nums, int k) {
    unordered_map<int,int> freq;
    for (int n : nums) freq[n]++;

    vector<vector<int>> buckets(nums.size() + 1);
    for (auto& [num, cnt] : freq) buckets[cnt].push_back(num);

    vector<int> res;
    for (int i = (int)buckets.size()-1;
         i >= 0 && (int)res.size() < k; i--)
        for (int num : buckets[i])
            if ((int)res.size() < k) res.push_back(num);
    return res;
}
use std::collections::HashMap;

pub fn top_k_frequent(nums: Vec<i32>, k: i32) -> Vec<i32> {
    let mut freq: HashMap<i32, usize> = HashMap::new();
    for &n in &nums { *freq.entry(n).or_insert(0) += 1; }

    let mut buckets: Vec<Vec<i32>> = vec![vec![]; nums.len() + 1];
    for (&num, &cnt) in &freq { buckets[cnt].push(num); }

    let mut result = Vec::new();
    for bucket in buckets.iter().rev() {
        for &num in bucket {
            result.push(num);
            if result.len() == k as usize { return result; }
        }
    }
    result
}
public int[] TopKFrequent(int[] nums, int k) {
    var freq = new Dictionary<int, int>();
    foreach (int n in nums)
        freq[n] = freq.GetValueOrDefault(n, 0) + 1;

    var buckets = new List<int>[nums.Length + 1];
    foreach (var (num, cnt) in freq) {
        buckets[cnt] ??= new List<int>();
        buckets[cnt].Add(num);
    }
    var result = new List<int>();
    for (int i = buckets.Length - 1; i >= 0 && result.Count < k; i--)
        if (buckets[i] != null) result.AddRange(buckets[i]);
    return result.Take(k).ToArray();
}
function topKFrequent(nums: number[], k: number): number[] {
    const freq = new Map<number, number>();
    for (const n of nums) freq.set(n, (freq.get(n) ?? 0) + 1);

    const buckets: number[][] =
        Array.from({ length: nums.length + 1 }, () => []);
    for (const [num, cnt] of freq) buckets[cnt].push(num);

    const result: number[] = [];
    for (let i = buckets.length - 1; i >= 0 && result.length < k; i--)
        result.push(...buckets[i]);
    return result.slice(0, k);
}

Longest Consecutive SequenceLC 128

HardHash SetChain Walk
Naïve
O(n log n)
O(1) space
→
Optimal
O(n)
O(n) space
Key insightLoad all elements into a hash set. A number n is a sequence start iff n−1 is absent. Only walk chains from their starts. Because each element is visited as a start at most once and walked at most once, the total work across all chains is O(n)—not O(n²) as it superficially appears.

Problem Statement

Given an unsorted array of integers nums, return the length of the longest consecutive elements sequence. You must write an algorithm that runs in O(n) time.

Example

Input
[100,4,200,1,3,2]
6 elements
Output
4
sequence: 1,2,3,4

Approach

Load all elements into a hash set in O(n). For each number n, check whether n−1 is in the set. If it is, n is not a sequence start—skip it. If it is not, n begins a new sequence: walk forward (n+1, n+2, ...) until you hit a gap, counting the length.

The amortised O(n) argument: every element is "started" at most once (only when it is a sequence start) and "walked" at most once (by its start). Across all sequences, total walk steps = total elements = O(n).

The hard rating reflects that the O(n) requirement rules out sorting, and the amortised argument for the chain-walk is non-obvious at first encounter.

func longestConsecutive(nums []int) int {
    set := make(map[int]bool)
    for _, n := range nums { set[n] = true }

    best := 0
    for n := range set {
        if !set[n-1] { // only walk from sequence starts
            length := 1
            for set[n+length] { length++ }
            if length > best { best = length }
        }
    }
    return best
}
public int longestConsecutive(int[] nums) {
    Set<Integer> set = new HashSet<>();
    for (int n : nums) set.add(n);

    int best = 0;
    for (int n : set) {
        if (!set.contains(n - 1)) {       // sequence start
            int length = 1;
            while (set.contains(n + length)) length++;
            best = Math.max(best, length);
        }
    }
    return best;
}
int longestConsecutive(vector<int>& nums) {
    unordered_set<int> set(nums.begin(), nums.end());

        if (!set.count(n - 1)) {           // sequence start

        }
    }
    return best;
}

        if !set.contains(&(n - 1)) {      // sequence start

        }
    }
    best
}

  • Count distinct sequences — count the number of sequence-start nodes.

Key insightDefine prefix[i] = nums[0]+…+nums[i−1]. A subarray nums[i..j] sums to k iff prefix[j+1] − prefix[i] = k, i.e. prefix[i] = prefix[j+1] − k. Maintain a frequency map of prefix sums seen so far; each new prefix sum queries "how many prior sums equal current − k?" This is the prefix-sum complement pattern, the subarray analogue of Two Sum.

Problem Statement

Given an array of integers nums and an integer k, return the total number of subarrays whose sum equals k.

Example

nums
[1,1,1]
k = 2
Output
2
subarrays: [1,1] at idx 0-1 and 1-2

Approach

Define prefix[i] = nums[0] + … + nums[i−1] with prefix[0] = 0. Any subarray nums[i..j] (inclusive) sums to k iff prefix[j+1] − prefix[i] = k, i.e. prefix[i] = prefix[j+1] − k.

As we scan, we maintain a running prefix sum and a frequency map of all prefix sums seen so far (seeded with {0: 1} to account for subarrays starting at index 0). At each position, we add prefixCount[currentSum − k] to our answer, then record the current prefix sum. This is structurally identical to Two Sum: "how many prior states give us what we need right now?"

Important: this problem does not require non-negative numbers (unlike the sliding-window technique). Negative values are handled correctly because we count all prefix sums, not just monotone ones.

func subarraySum(nums []int, k int) int {
    // prefixCount[x] = how many times prefix sum x has occurred
    prefixCount := map[int]int{0: 1}
    sum, count := 0, 0
    for _, n := range nums {
        sum += n
        count += prefixCount[sum-k]
        prefixCount[sum]++
    }
    return count
}
public int subarraySum(int[] nums, int k) {
    Map<Integer, Integer> prefixCount = new HashMap<>();
    prefixCount.put(0, 1);
    int sum = 0, count = 0;
    for (int n : nums) {
        sum += n;
        count += prefixCount.getOrDefault(sum - k, 0);
        prefixCount.merge(sum, 1, Integer::sum);
    }
    return count;
}
int subarraySum(vector<int>& nums, int k) {
    unordered_map<int,int> prefixCount{{0,1}};

    }

use std::collections::HashMap;

pub fn subarray_sum(nums: Vec<i32>, k: i32) -> i32 {
    let mut prefix_count: HashMap<i32, i32> = HashMap::new();
    prefix_count.insert(0, 1);
    let (mut sum, mut count) = (0, 0);
    for n in nums {
        sum += n;
        count += prefix_count.get(&(sum - k)).copied().unwrap_or(0);
        *prefix_count.entry(sum).or_insert(0) += 1;
    }
    count
}

function subarraySum(nums: number[], k: number): number {
    const prefixCount = new Map<number, number>([[0, 1]]);
    let sum = 0, count = 0;
    for (const n of nums) {
        sum += n;
        count += prefixCount.get(sum - k) ?? 0;
        prefixCount.set(sum, (prefixCount.get(sum) ?? 0) + 1);
    }
    return count;
}

Competition Tips

See nested loops → think hash

Any O(n²) with an inner linear scan is a candidate. Ask: what is the inner loop searching for? Can a hash map answer that in O(1)?

Frequency arrays for bounded inputs

If values are in [0,26) or [0,256), use arrays. Array access is 5–10× faster than hash map in practice.

Seed maps with a base case

Prefix-sum maps need {0:1} (or {0:-1} for index) as the base case before the first element.

Two-pointer beats hash for sorted

If the array is already sorted, two-pointer gives the same O(n) with O(1) space. Always check constraints first.

Canonical key = grouping superpower

Sort, frequency tuple, bit mask—any deterministic transformation of the object that collapses equivalences into identical strings/arrays.

Bucket sort when range ≤ n

TopK, sort by frequency, sort by length: if the "key" is bounded by n, buckets beat O(n log n) comparison sort.

Count complement maps

Subarray sum, Two Sum, Range Sum: reformulate as "how many prior states satisfy current − target = state?" Store counts, not just presence.

Amortised ≠ worst-case

Hash map O(1) is amortised. For hard real-time constraints consider sorted maps (O(log n) guaranteed) or open-addressing implementations.
