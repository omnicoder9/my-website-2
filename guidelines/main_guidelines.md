site guidelines - rules to be followed during agentic development tasks (that means you).
0. no surprises - no out of scope changes. no high cost or high risk changes without confirmation.  be a good friend.
1. all md files in the guidelines directory should be mentioned in main_guidelines.md at least once.  If this is not the case, report this as "WARNING: ISOLATED MD FILE IN GUIDELINES: *name of md file here*.md".
2. all categories in the blog should be listed in blog-categories.md.  if a category is deleted from the blog, it should also be erased from blog-categories.md and vice-versa.
3. all blog articles should be labeled with at least one category.  if a blog article does not fit any category or would benefit from a new one, suggest it and ask for approval.
